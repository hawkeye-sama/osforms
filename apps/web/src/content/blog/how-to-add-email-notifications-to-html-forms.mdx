Every web app has at least one form. Contact forms, waitlist signups, feedback widgets. And every time you ship one, you need the same thing: **get notified when someone fills it out**.

The usual path for HTML form email notifications is painful. You stand up a server, wire up an email provider, handle errors, manage retries. Or you sign up for Formspree and hit a paywall the moment you want email forwarding — even though you're supplying the Resend API key yourself.

There's a better way.

## What You'll Build

A plain HTML contact form that:

- Stores every submission in a dashboard
- Sends email notifications via your own Resend account
- Requires zero backend code from you

> "You're not paying for integrations. You're paying for the markup on your own API keys."

## Prerequisites

- An [osforms account](https://osforms.com/signup) — free, no card required
- A [Resend account](https://resend.com) — free tier covers 3,000 emails/month

## Step 1 — Create a Form in osforms

Log in and hit **New Form**. Give it a name like "Contact Form". You'll get a unique endpoint URL:

```bash
https://osforms.com/api/v1/f/abc123xyz
```

Copy that URL — you'll use it as your form's `action` attribute. That endpoint is all you need. No servers. No middleware.

## Step 2 — Connect Your Resend API Key

In the dashboard, go to **Integrations → Email**. Paste your Resend API key (starts with `re_`). Enter the "from" address and the "to" address where notifications should land.

osforms encrypts your key at rest with AES-256-GCM. It's never stored in plain text, never logged, and only decrypted in memory when a submission arrives.

> "Your API key powers the email. osforms just makes the call."

## Step 3 — Point Your Form at the Endpoint

Update your HTML form's `action` attribute:

```html
<form action="https://osforms.com/api/v1/f/abc123xyz" method="POST">
  <label for="name">Name</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email</label>
  <input type="email" id="email" name="email" required />

  <label for="message">Message</label>
  <textarea id="message" name="message" rows="4" required></textarea>

  <button type="submit">Send</button>
</form>
```

That's it. Submit the form and within seconds you'll get an email. The form works on any stack — plain HTML, React, Vue, Astro, anything that can send a POST request.

## Step 4 — Set a Success Redirect

By default, after a submission, osforms redirects back to the referring page. Set a custom redirect URL in your form settings — point it to a `/thank-you` page for a cleaner experience.

## How It Works Under the Hood

When a submission hits your endpoint:

1. osforms stores it in the dashboard immediately
2. Returns a `200 OK` — your visitor never waits
3. Your Resend integration fires asynchronously in the background via `waitUntil()`
4. The email arrives with all submitted fields formatted and readable

Integrations run in parallel using `Promise.allSettled()`. If Resend is unavailable, the submission is still stored and you get an in-app notification — nothing is lost.

## Spam Protection

Two built-in options that require zero configuration on your end:

- **Honeypot field**: Add a hidden input named `_honey`. Bots fill it in; osforms silently discards those submissions.
- **reCAPTCHA / hCaptcha / Turnstile**: Bring your own site key. osforms verifies the token server-side before storing anything.

## FAQ

**Does this work with React and Next.js, not just plain HTML?**
Yes. You can use the standard HTML `<form>` tag in any framework, or use `fetch` / `axios` to POST JSON to your endpoint. osforms handles both `application/x-www-form-urlencoded` and `application/json` content types.

**What happens if Resend is down when a submission arrives?**
The submission is stored immediately regardless. If the Resend call fails, you get an in-app notification in the osforms dashboard with the error. You can review the submission and resend manually.

**Can I send to multiple email addresses?**
Set up multiple email integrations pointing to different addresses, or configure your Resend account to forward to a distribution list. osforms runs all configured integrations in parallel for every submission.

**Is the `_honey` honeypot field visible to users?**
No. You add it as a hidden field with CSS (`display: none` or `visibility: hidden`). Real users never see or fill it. Bots that auto-fill all fields get caught by the mismatch.

**What's the difference between osforms and just using Resend directly?**
Resend requires you to write server-side code — an API route, a serverless function, something that runs on your infrastructure. osforms is the server code. You point your HTML form at our endpoint, and we handle the server layer, storage, and Resend call. Zero backend required on your side.
